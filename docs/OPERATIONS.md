# 运维、内容治理与质量控制

## 日常看板

至少监控：

- 各数据源最后成功时间、错误率、延迟和剩余配额。
- 候选事件数、合并率、热点通过率和误报率。
- 从首次发现到发布的 P50/P95 时间。
- 事实置信度、人工驳回率、修改率、纠错和撤稿率。
- 各账号日发布数、失败率、冷却状态和互动转化。
- X、LLM、对象存储等按日/按月费用与预测。

发布队列默认由定时 Worker 消费；本地或人工补偿可调用 `POST /api/publications/run`。运行器必须只领取到期任务并使用数据库锁/租约，避免多个实例并发发布同一任务。当前实现为 dry-run Mock 发布，正式适配器上线前不要把模拟成功等同于平台送达。

## 内容质量门

发布前必须满足：

- 至少一个可访问来源；高风险新闻至少一个一手来源或两个独立可靠来源。
- 事实、推测和观点显式分开。
- 数字包含币种、单位、时间窗口和数据时间。
- 截图保留主体、发布时间和必要上下文，不暴露个人敏感信息。
- 标题不把“可能”“据称”改成已确认事实。
- 不包含收益承诺、确定性价格预测和个性化买卖建议。

## 视觉 QA

每个视觉产物需通过自动与人工抽检：

- 输出尺寸、文件类型、体积和最多四图限制。
- 中文字体加载成功，无截断、重叠、乱码或小于移动端可读字号。
- 信息卡上的金额、涨跌幅和时间与 Fact pack 一致。
- 证据截图记录 URL、抓取时间、视口和内容哈希。
- 第三方 Logo、照片和媒体截图符合使用条款；无法确认时使用自有信息卡。

截图失败不能阻塞事实入库，可退化为品牌信息卡；但不能生成看似原始证据的仿造截图。

## 安全事件响应

### Key 泄漏

1. 关闭对应账号发布开关。
2. 撤销并轮换 Key。
3. 检查该账号近期待发布和已发布审计记录。
4. 清除日志、缓存或 CI 中的意外副本。
5. 记录根因和防复发措施。

### 错误信息发布

1. 停止同一事件的所有后续分发。
2. 固化当时的事实包、模型输入、审核和发布记录。
3. 由编辑决定更正、澄清或平台人工删除。
4. 更新来源可信度或规则，不能只修改最终文案。

### 成本异常

1. 自动熔断付费增强源。
2. 检查分页循环、缓存失效、事件未过期和重试风暴。
3. 免费源继续工作，候选标记为“未增强”。

## 数据保留

建议：原始抓取按来源条款设置生命周期；事实包、发布版本、审核和审计日志长期保存；带个人信息的内容最小化保存并支持删除请求。上线前应由熟悉目标司法辖区的专业人士审查版权、隐私、平台条款和金融内容规范。
