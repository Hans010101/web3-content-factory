# 数据源接入指南

## 免费优先的来源组合

| 优先级 | 来源 | 用途 | 注意事项 |
|---|---|---|---|
| P0 | 项目/交易所/监管官方 RSS 与公告 | 一手确认 | 保存原始链接和发布时间 |
| P0 | Binance 公开市场 REST/WebSocket | 价格、成交量、异动 | 限流、断线重连、时间同步 |
| P1 | GitHub Releases/Commits | 升级、发布、安全修复 | 只监控精选仓库，避免噪声 |
| P1 | DefiLlama 等公开数据 | TVL、稳定币和协议变化 | 明确数据口径与延迟 |
| P1 | GDELT | 全球媒体候选线索 | 聚合结果不能单独作为事实 |
| P1 | 公开网页监控 | 无 RSS 的官网、博客和长文 | Jina Reader 优先、原网页直读回退，只允许公开 URL |
| P2 | 公链 RPC/浏览器免费额度 | 大额转账、合约事件 | 地址归因必须二次确认 |
| P2 | YouTube/Wikipedia/HN | 辅助判断关注趋势 | 只作为热度侧信号 |
| P3 | X API | 早期讨论、KOL观点 | 付费增强，严格预算上限 |

## 接入标准

每个适配器必须输出：

```json
{
  "sourceId": "official-rss",
  "externalId": "provider-stable-id",
  "canonicalUrl": "https://example.com/item",
  "publishedAt": "2026-07-14T08:00:00Z",
  "collectedAt": "2026-07-14T08:01:12Z",
  "title": "原始标题",
  "body": "原始正文或摘要",
  "language": "zh",
  "entities": ["BTC"],
  "contentHash": "sha256:...",
  "rawRef": "object-storage-key"
}
```

要求：

- `externalId + sourceId` 唯一，支持重复轮询而不产生重复记录。
- 时间统一保存 UTC，UI 可按 Asia/Singapore 展示。
- 保留原始响应到有生命周期策略的对象存储，数据库只保存引用。
- 记录 HTTP 状态、配额头、游标和适配器版本。
- HTML 清洗后仍保留原始链接；不可绕过付费墙或访问控制。

## 分阶段采集

### 阶段 A：低成本发现

持续读取官方 Feed、市场异动、GDELT、GitHub 与公开链上信号，按关键词和资产白名单生成候选事件。此阶段只保存必要字段并快速去重。

### 阶段 B：证据增强

只有超过候选阈值的事件才追加：官方页面截图、更多市场窗口、链上交易详情和 X 讨论样本。每个候选设最大请求数、最大数据条数和美元预算。

### 阶段 C：持续追踪

对已发布且仍发展的事件设置有限的追踪窗口。出现官方更正、损失金额变化或市场冲击升级时生成后续版本，过期后停止轮询。

## X 的成本护栏

X 默认关闭。启用前设置：

- 每月美元上限。
- 每日最多增强候选数。
- 每个候选最多读取条数和页数。
- 当日相同帖子/用户缓存。
- 达到 80% 预算报警，100% 自动熔断。

不要用 MCP 或 Agent 自由分页作为生产采集器；固定采集使用 REST API，MCP 适合运营临时调研。

## 来源白名单

真实接入时维护版本化白名单：官方域名、RSS URL、GitHub 仓库、链 ID、合约地址、账号 ID。禁止只按显示名称判断“官方”，域名跳转和账号改名需人工审核。

建议为来源设置：

- `role`: `discover` 或 `confirm`。
- `trust`: 历史可信度，不等同于单条事实置信度。
- `topics`: 用于候选路由。
- `costTier`: `free`、`metered` 或 `contract`。
- `termsReviewedAt`: 最近一次审查使用条款的日期。

## 故障与降级

- 单一来源失败：保留最后检查点，其他来源继续工作。
- 市场流断线：重连后用 REST 补齐缺口，避免把缺口当成成交量异常。
- 配额耗尽：停止付费增强，不影响免费源发现。
- 时间戳异常：隔离该批数据，禁止进入速度评分。
- 来源内容被删除：保留审计元数据，按使用条款处理原始副本，不继续公开展示。

## 诊断与后端路由

每个来源维护有序的后端候选列表。体检必须真实请求轻量端点，而不是只判断 Key、URL 或程序是否存在；第一个健康后端成为当前后端，首选失败后才允许降级。

- T0：无需用户配置即可运行。
- T1：需要免费 Key、RPC、仓库或 URL 清单。
- T2：需要付费套餐、平台审批或高摩擦授权。
- 诊断结果缓存 60 秒，手动“重新体检”可强制绕过缓存。
- 每个来源独立捕获异常，任何一个探测崩溃都不能让整份诊断失败。
- GitHub 认证请求遇到 401/403 时自动回退匿名只读；匿名配额耗尽后停止，不循环重试。
- 公开网页使用内容指纹去重，最多并行读取 10 个 URL，单页限制 2MB。

不在服务器中复用运营人员的主账号 Cookie。X、Reddit、小红书等登录态采集如果未来引入，必须使用专用小号、独立会话、条款审查与明确的封号风险提示。
